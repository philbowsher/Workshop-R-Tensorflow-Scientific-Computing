# Keras + Vetiver: Peptide Binding Prediction

Complete ML deployment pipeline using **Keras** and **Vetiver** for MLOps.

## Files

### 1_train_model.R
Train the peptide binding prediction model (identical to part_3 training).

**Output:** `peptide_model_keras.keras`

---

### 2_version_model_vetiver.R
Create a Vetiver model object and pin to Connect.

**What it does:**
```r
# Create vetiver model with metadata
v <- vetiver_model(
  model = model,
  model_name = "peptide_keras_vetiver",
  description = "Peptide-MHC binding prediction (Keras)",
  metadata = list(
    framework = "keras",
    classes = c("NB", "WB", "SB"),
    accuracy = "~95%"
  )
)

# Pin to Connect (with automatic versioning)
vetiver_pin_write(board, v)
```

**What you get:**
- Model card with metadata
- Automatic versioning
- Ready for deployment

---

### 3_deploy_api_vetiver.R
Deploy the API to Connect (automatically generated by Vetiver).

**What it does:**
```r
# This ONE line deploys the entire API!
vetiver_deploy_rsconnect(
  board = board,
  name = "peptide_keras_vetiver"
)
```

**What you get:**
- Auto-generated plumber.R
- `/predict` endpoint
- `/ping` endpoint (health check)
- `/metadata` endpoint (model info)

**No manual plumber.R file needed!**

---

### 4_consume_api.R
Call the deployed Vetiver API.

**Key differences from manual API:**
- POST request (not GET)
- JSON body (not query params)
- Returns raw probabilities (not class labels)

**Example:**
```r
# Create endpoint
api_endpoint <- vetiver_endpoint("https://pub.workshop.posit.team/content/abc-123")

# Prepare data (encoded peptides as data frame)
input_df <- as.data.frame(encoded_peptides)

# Make predictions
predictions <- predict(
  api_endpoint,
  input_df,
  httr::add_headers(Authorization = paste0("Key ", api_key))
)
```

## Complete Workflow

```r
# Step 1: Train model
setwd("part_5_vetiver/tensorflow_keras")
source("1_train_model.R")

# Step 2: Version with Vetiver
Sys.setenv(CONNECT_SERVER = "https://pub.workshop.posit.team")
Sys.setenv(CONNECT_API_KEY = "your-api-key")
source("2_version_model_vetiver.R")

# Step 3: Deploy API
source("3_deploy_api_vetiver.R")

# Step 4: Get endpoint from Connect UI
# Go to: https://pub.workshop.posit.team
# Find: "Peptide Prediction API (Keras + Vetiver)"
# Copy URL: /content/abc-123/

# Step 5: Consume API
Sys.setenv(VETIVER_ENDPOINT = "content/abc-123")
source("4_consume_api.R")
```

## API Endpoints

### POST /predict
Make predictions

**Request:**
```json
POST /predict
Authorization: Key your-api-key
Content-Type: application/json

{
  "V1": 0.5, "V2": 0.3, ..., "V180": 0.1
}
```

**Response:**
```json
[
  [0.1, 0.7, 0.2],  // Peptide 1: [NB_prob, WB_prob, SB_prob]
  [0.05, 0.15, 0.8] // Peptide 2: [NB_prob, WB_prob, SB_prob]
]
```

---

### GET /ping
Health check

**Request:**
```bash
GET /ping
```

**Response:**
```json
{"status": "ok"}
```

---

### GET /metadata
Model information

**Request:**
```bash
GET /metadata
Authorization: Key your-api-key
```

**Response:**
```json
{
  "model_name": "peptide_keras_vetiver",
  "description": "Peptide-MHC binding prediction (Keras)",
  "framework": "keras",
  "classes": ["NB", "WB", "SB"],
  "version": "2024-05-05T12:34:56Z"
}
```

## Troubleshooting

### Error: Model not found
**Solution:** Run step 2 first to version/pin the model

### Error: Unauthorized (401)
**Solution:** Set `CONNECT_API_KEY` environment variable

### Error: Endpoint not found (404)
**Solution:** Get correct endpoint from Connect UI after deployment

### Error: Model predictions wrong format
**Solution:** Ensure input is a data frame with 180 columns (V1-V180)

## Production Tips

1. **Versioning**: Every `vetiver_pin_write` creates a new version
   ```r
   # View versions
   pin_versions(board, "peptide_keras_vetiver")
   
   # Load specific version
   v <- vetiver_pin_read(board, "peptide_keras_vetiver", version = "20240505T123456Z")
   ```

2. **Monitoring**: Use `/ping` for health checks in CI/CD
   ```r
   # Automated health check
   response <- httr::GET(paste0(api_url, "/ping"))
   if (response$status_code != 200) {
     # Alert ops team
   }
   ```

3. **A/B Testing**: Deploy multiple versions, compare performance
   ```r
   # Deploy v1
   vetiver_deploy_rsconnect(board, "model_v1", appTitle = "API v1")
   
   # Deploy v2
   vetiver_deploy_rsconnect(board, "model_v2", appTitle = "API v2")
   
   # Route traffic to test
   ```

4. **Model Cards**: Document your model
   ```r
   v <- vetiver_model(
     model,
     "model_name",
     metadata = list(
       training_date = Sys.Date(),
       dataset = "mhc_igh",
       accuracy = 0.95,
       author = "Data Science Team",
       notes = "Production model v2.1"
     )
   )
   ```
